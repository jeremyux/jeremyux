Styleguide & Components Library

Google

In a fast design enviroment where formats are likely to have inconsistencies, we needed a tool where we can take inventory of all of our different ads formats.

Our formats constantly change as we hyper-optimize them based on experiment metrics. It's difficult to keep track of what is currently in production.

Additionally we are in the process of hiring new designers to the team. At the time onboarding new designers was a difficult and lengthy process. I wanted this component library to be easily digestable and ready to implement into their mocks.

Component Library
We wanted this component library to not only match what we have in production but also be able to be updated easily. The goal is to drag and drop Sketch symbols from the component library to working project files.

Text Field and Customization
We design ads that contantly use real data. As a designer I would want all the fields ready to be updated so I can use them in my mocks. Engineers and Project Managers also found this tool useful for conveying out-of-the-box ideas through mocks.

Color Palette & Type Ramp
In the past, designers new to the team learned the team's design patterns through outdated decks, past project files and using the browser's inspect mode. Now we have one source of truth for everyone to refer to. Here we included an inventory of color and typography across all of our different ads formats so we can quickly refer to them.
