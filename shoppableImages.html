Shoppable Images

Google

60% of queries on Image Search are "shoppy" by nature. From the merchant companies' perspectives, Google are losing on opportunities to surface product ads to our users. How might we better serve our users' shopping intents as well as better utilize our merchant data to create an elegant e-commerce experience on Image Search?

Ads were already present on Image Search at the beginning of this project. The only ads present were in the form of a horizontal carousel that sat at the top of the search page. The restrictions of this format were that users had to scroll horizontally, which classically returns poor metrics in terms or interaction and click-through-rate. So when forming the design basis to this solution, we had to think about how the user interacts with the page overall.

The Short Answer:
Users scroll vertically. By leveraging this we can give an elegant solution in respect to the canvas it lives on.

Pilot: The Behavior
As the user scrolls, the ad will become 40% viewable in the viewport. This event will trigger the "offer bubbles". These "offer bubbles" will serve as an entrypoint to a merchant's product landing page. There is also a toggle that will dismiss said "offer bubbles" which will give the user a clear view of the image. An Image Search best practice is to always give the user the ability to see the image in full view.

Alpha: 2nd Iteration
We discovered some findings when we tested our new Shoppable Images Format on a small number of users. There seems to be an issue with engagement as numbers were low. Maybe we can differentiate our unit as opposed to looking like an organice result on this page.

We decided to not only cardify this unit but also bring a branding flavor to our ad to bring both familiarity and association to the brands.

Alpha: 3rd Iteration
Visually the card not only looks enticing but also clearly looks like an ad. Though now we are running into a problem where users are clicking into the homepage and immediately going back to Image Search's search results page. Our base assumption is that users are confused about where they have landed because they to not see the image that was used in the entrypoint. Merchants don't actually supply us with the image in the ad.

The Remedy
To fix this problem we decided to change the link of the image click itself. Sending them to the homepage would confuse users, so why don't we use this opportunity to showcase the offers underneath the unit itself? By doing so we are not only able to give extra affordance to the offers we present, but also provide extra value to the user.

Where We Are With This Project
This project is now live, supporting certain shopping queries. Since this has gone live, contributors of this project have received a patent for this new format, including myself.
